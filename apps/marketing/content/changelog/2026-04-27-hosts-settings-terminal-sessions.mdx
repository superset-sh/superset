v2 is in public beta. Enable it under **Settings → Experimental**. <PRBadge url="https://github.com/superset-sh/superset/pull/3748" /> v1 projects and workspaces port directly into v2, projects can now be created and configured end-to-end inside v2, the sidebar grew a hover card with AI rename and PR-state icons, terminal sessions span workspaces, and chat picked up GPT-5.5 and Opus 4.7.

{/* TODO: Add Settings → Experimental screenshot showing the v2 toggle */}

## v1 → v2 migration is now real

v1 workspaces and projects can be ported into v2 in place — no manual recreation, no losing branch state. <PRBadge url="https://github.com/superset-sh/superset/pull/3670" />

- **Stabilized migration** with org-level idempotency so repeat runs don't double-up workspaces <PRBadge url="https://github.com/superset-sh/superset/pull/3775" /> <PRBadge url="https://github.com/superset-sh/superset/pull/3781" /> <PRBadge url="https://github.com/superset-sh/superset/pull/3783" />
- **Migration modal only fires on first onboard** instead of every launch <PRBadge url="https://github.com/superset-sh/superset/pull/3816" />
- **v1 keypad loading screen ported to v2** as a dedicated page <PRBadge url="https://github.com/superset-sh/superset/pull/3788" />

## v2 project setup, end-to-end

You can now create and import projects entirely in v2 — no falling back to v1 to set things up.

- **Create or import a project** straight from v2 <PRBadge url="https://github.com/superset-sh/superset/pull/3566" />
- **Project name inferred from the folder** on import <PRBadge url="https://github.com/superset-sh/superset/pull/3605" />
- **v2 project settings** with a setup/relocate path <PRBadge url="https://github.com/superset-sh/superset/pull/3606" /> wired into the sidebar settings route <PRBadge url="https://github.com/superset-sh/superset/pull/3592" />
- **Non-setup projects route to settings** from the new-workspace modal so you finish setup in place <PRBadge url="https://github.com/superset-sh/superset/pull/3626" />
- **Auto-create a v2 main workspace** when a host project is set up <PRBadge url="https://github.com/superset-sh/superset/pull/3632" />

## Scheduled agent runs <PRBadge url="https://github.com/superset-sh/superset/pull/3576" />

Automations now run on a schedule, end-to-end. Pick a cadence, pick an agent and prompt, and it fires on its own — same execution path as a manual run.

## Cross-workspace terminal sessions <PRBadge url="https://github.com/superset-sh/superset/pull/3751" />

The terminal session dropdown in v2 lists sessions from all your workspaces, grouped by workspace. Selecting a session in another workspace navigates there with the terminal focused. Sessions show their live shell or process title so you can tell them apart at a glance. <PRBadge url="https://github.com/superset-sh/superset/pull/3740" /> Sessions from other workspaces can be hidden if you'd rather scope it to the current one. <PRBadge url="https://github.com/superset-sh/superset/pull/3774" />

## Mark workspace as unread <PRBadge url="https://github.com/superset-sh/superset/pull/3773" />

Right-click any v2 sidebar workspace and **Mark as Unread** — it gets the same green dot as a real agent-ready notification. Clicking the workspace clears it automatically <PRBadge url="https://github.com/superset-sh/superset/pull/3765" />, and v2 notification hooks now play client-side so the sound fires reliably <PRBadge url="https://github.com/superset-sh/superset/pull/3675" />.
