## Model Picker in New Workspace Dialog <PRBadge url="https://github.com/superset-sh/superset/pull/5248" />

You can now select an AI model directly when creating a new workspace — no need to edit CLI configs or restart with different flags. The picker appears next to the agent selector and supports Claude (Opus, Sonnet, Haiku), Codex, Gemini, Copilot, Cursor Agent, and OpenCode. Your last-used model per agent is remembered across sessions.

- Choose **Default** to let the agent use its own configured model
- Per-agent model preference is persisted and restored when you reopen the dialog
- Model selection is injected into the agent launch command (e.g. `claude --model sonnet`)

## Command Palette Search Button <PRBadge url="https://github.com/superset-sh/superset/pull/5346" />

A magnifying-glass button now appears in the dashboard navigation bar, next to the Recently Viewed button. It opens the command palette — the same as `Cmd+Shift+K` — making it more discoverable without a keyboard shortcut.

## Improvements

- **Paste images into new workspace prompt** - Screenshots and images pasted from the clipboard now appear as attachment pills in the new workspace creation dialog <PRBadge url="https://github.com/superset-sh/superset/pull/5283" />
- **Faster terminal rendering** - PTY output is now batched per animation frame instead of one write per WebSocket chunk, reducing CPU overhead and improving smoothness for agent CLIs like Claude Code <PRBadge url="https://github.com/superset-sh/superset/pull/5255" />
- **Quieter git watcher** - `.git` directory events are filtered out and debounce is now adaptive, reducing background noise from the file watcher <PRBadge url="https://github.com/superset-sh/superset/pull/5239" />
- **Settings search covers Scripts section** - The project setup/teardown scripts section now appears in settings search results <PRBadge url="https://github.com/superset-sh/superset/pull/5335" />
