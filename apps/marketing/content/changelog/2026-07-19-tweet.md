What we shipped this week @superset_sh 🛳️

1. Running terminals come back when you open a workspace ♻️
Sessions started by an automation, a host-side launch, or your last app run used to hide in the background-terminals dropdown until you adopted them one at a time. That's silly. Now opening a workspace just recreates their panes, so you see what's actually running. The dropdown still holds anything you deliberately send to the background.

2. Agents that fail now say so 🔴
A failed agent turn used to sit on "working" forever. Claude Code now reports API errors like rate limits, overloads, and auth failures as a red "failed" status on the pane, and it counts toward your unread and dock badge. It clears itself when the agent retries.

3. Find in the Changes pane 🔍
Press ⌘F in a workspace Changes pane and search the whole diff, not just the rows on screen. Next/previous jumps to matches in collapsed files and expands them for you.

4. Confirm before closing a running terminal ⌨️
⌘W used to silently kill whatever was running. Now you get a prompt only when a command is actually running, with a "Don't ask again" checkbox. Idle prompt still closes instantly.

Also this week: a terminal connection popover that tells you why a pane dropped, Clear Status now unsticks a wedged agent, adding an agent auto-creates its launcher everywhere, better auto-named workspaces when you don't run Claude or Codex, and refreshed Cursor models.

This changelog draft was created by an automation.

Plus a bunch of bug fixes. Thanks to everyone in the v2 open beta for the reports, please keep them coming!
