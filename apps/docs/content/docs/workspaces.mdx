## Core Concept

Each workspace = one branch. Workspaces are isolated environments with their own directory, terminal, and ports. Superset uses git worktrees for Git repos and native jj workspaces for Jujutsu repos.

## Create Workspaces

Press **⌘N** or click "New Workspace". Create from:

- **New branch** - Fresh branch from main
- **Existing branch** - Continue work on a branch
- **Pull request** - Review or continue a PR

Each workspace gets:
- Own branch (git branch or jj bookmark)
- Own working directory
- Own terminal sessions
- Own port range

## Jujutsu (jj) Support

Superset auto-detects Jujutsu repos in colocated mode (`.jj` + `.git` side by side) and uses native jj commands:

- **Workspace creation** uses `jj workspace add` instead of `git worktree add`
- **Bookmarks** are listed instead of git branches in the branch selector
- **jj commands work** inside workspaces (auto-rebasing, `jj log`, etc.)
- **Changes UI** (staging, commits, push) continues to work via git in colocated mode

No configuration needed — just open a jj colocated repo and Superset handles the rest. If the `jj` CLI is not installed, Superset falls back to git.

## Use Cases

- **Parallel features** - Work on multiple features simultaneously
- **AI agents** - Run Claude Code in separate workspaces
- **Code review** - Review PRs while continuing your work

## Best Practices

- Use descriptive branch names: `feature/auth`, `fix/login-bug`
- Don't work directly on `main`
- Delete workspaces after merging
