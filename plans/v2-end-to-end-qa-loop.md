# V2 End-to-End QA Loop

This plan defines a v2-only QA loop for regular user workflows across the
desktop app, host service, CLI, MCP surface, and docs-backed feature set. The
goal is to find latent bugs that users hit while adding repositories, creating
workspaces, running panes and tabs, reviewing changes, launching agents, and
recovering from common failures.

## Scope

In scope:

- V2 projects/repositories, V2 workspaces, and host-service-backed workspace
  operations.
- Desktop routes under
  `apps/desktop/src/renderer/routes/_authenticated/_dashboard/v2-workspace`.
- Dashboard repository, project, task, automation, settings, and sidebar flows
  that launch or manage V2 workspaces.
- Host service routers for projects, workspace creation, workspace cleanup,
  git/diff, filesystem, terminal, ports, chat, agents, issues, pull requests,
  and automations.
- CLI and MCP v2 workflows where docs describe project, workspace, automation,
  task, host, and agent operations.

Out of scope:

- V1 workspace creation and V1 panes except where a shared component can break
  V2. The existing `apps/desktop/docs/V2_LAUNCH_TEST_PLAN.md` has one V1
  regression check, but this loop keeps V2 as the product scope.
- Production database writes. All fixture and stress work must use local dev
  state, preferably a disposable `SUPERSET_HOME_DIR`.
- Schema migrations.

## Test Data Model

Use a clean local home for destructive or high-volume runs:

```bash
export SUPERSET_HOME_DIR="/tmp/superset-v2qa-home"
export SKIP_ENV_VALIDATION=1
```

Create or seed these repo shapes:

- `simple-js`: small git repo with one package, one README, one changed file.
- `monorepo`: nested `apps/`, `packages/`, package manager files, and
  `.superset/config.json`.
- `large-diff`: fixture generated by
  `bun --cwd apps/desktop stress:renderer:fixtures`.
- `existing-worktrees`: local repo with multiple git worktrees already on disk.
- `remote-like`: repo with GitHub `origin`, branches, and PR/issue metadata when
  auth is available.
- `offline-local`: local repo with no GitHub remote.
- `edge-paths`: spaces, unicode-adjacent ASCII-safe names, nested paths, binary
  files, large files, symlinks if supported, deleted files, renames, and
  conflict-prone branch names.

## Journey Matrix

### 1. Repository And Project Onboarding

Primary happy paths:

- Add repository from a GitHub HTTPS URL.
- Add repository from a local path to an existing checkout.
- Add repository from a local path whose actual git root is above the selected
  folder.
- Add repository through CLI with `projects create --local --clone`.
- Add repository through CLI with `projects create --local --import`.
- Set up a cloud project on the current machine with `projects setup --local`.

Edge cases:

- Empty URL/path, malformed GitHub URL, SSH-style GitHub remote, local absolute
  path, relative path, and path with spaces.
- Destination parent does not exist or is not writable.
- Clone target already exists.
- Local path is not a git repo.
- Repo has no GitHub remote.
- Repo has multiple remotes and the expected remote is not `origin`.
- Existing local project row points at the path, but cloud project lookup fails.
- Cloud project was deleted while local rows remain.
- Same GitHub repo is represented by multiple V2 projects, but the same local
  path cannot be claimed by two projects.

Implementation areas:

- `NewProjectModal`
- `useFolderFirstImport`
- `packages/host-service/src/trpc/router/project/project.ts`
- `packages/host-service/src/trpc/router/project/utils/resolve-repo.ts`
- `packages/cli/src/commands/projects`

### 2. Workspace Creation

Primary happy paths:

- Create workspace from a new branch off the default branch.
- Create workspace from an existing local branch.
- Create workspace from a remote branch.
- Create workspace from a pull request by search result.
- Create workspace from a pull request by pasted PR URL.
- Create workspace with prompt-first launch and no attachments.
- Create workspace with prompt, issue, PR, task, and attachments.
- Create workspaces from CLI with `workspaces create --local`.
- Create workspaces from MCP v2 `workspaces_create`.

Edge cases:

- Branch name collision, invalid branch characters, very long prompt-derived
  branch names, empty prompt, whitespace-only prompt, and duplicate PR/issue
  links.
- GitHub auth missing, `gh` missing, network offline, PR not found, branch not
  found, and default branch missing.
- Setup script succeeds, fails, hangs, emits huge output, or writes files.
- Concurrent creation of two workspaces for different projects and the same
  project.
- Creation is interrupted by app quit and resumed on next launch.
- Workspace create row reaches failed state and retry is used.

Implementation areas:

- `packages/host-service/src/trpc/router/workspace-creation`
- `packages/host-service/src/trpc/router/project/utils/ensure-main-workspace.ts`
- `apps/desktop/src/shared/context`

- `apps/desktop/docs/V2_LAUNCH_TEST_PLAN.md`

### 3. Workspace Import, Delete, And Recovery

Primary happy paths:

- Import one discovered external worktree.
- Import all discovered external worktrees.
- Delete a normal branch workspace.
- Delete a PR workspace.
- Delete a project while preserving the main repo directory.
- Navigate away from a deleted active workspace to the next valid target.

Edge cases:

- Import duplicate worktree twice.
- Import worktree whose branch is detached or missing upstream.
- Worktree directory was manually removed before navigation.
- Teardown succeeds, fails, or requires force delete.
- `git worktree remove` fails during project deletion.
- Host service is offline during delete.
- Deleting active workspace while background terminal is still running.

Implementation areas:

- `listProjectWorktrees`
- `adoptExistingWorktree`

- `WorkspaceMissingWorktreeState`
- `DashboardSidebarDeleteDialog`

## Execution Phases

1. Static and unit coverage:
   - Run targeted tests for workspace creation, project resolution, git/diff
     surfaces, panes, terminal runtime, workspace navigation, and launch context.
   - Run lint on changed files or full lint before pushing fixes.

2. Host-service integration:
   - Run workspace create/delete integration.
   - Run git integration and v2 diff surface integration.
   - Run workspace cleanup integration.
   - Run recent bug-hunt integration.

3. Renderer stress:
   - Seed a small fixture first.
   - Start desktop with CDP enabled.
   - Run `workspace-switch`, `workspace-heavy`, `route-sweep`, and
     `terminal-heavy`.
   - Repeat with large fixture if the small run is clean.

4. Manual exploratory:
   - Use the journey matrix above as the checklist.
   - Record pass/fail and issue links in this plan.
   - Prefer one defect per issue with exact repro, expected result, actual
     result, logs, screenshots, and fixture identifiers.
